Week 5 Notes

Types of Errors:
System Error
Programmer Error
User Error

Exceptions - error that produces a return value that can then be used by the program to deal with an error; can be thrown using "throw" statement; produces error object

Stack Trace - produced by exception to point where the error occurred.

Warning - error in code that isn't enough to crash the program

Strict Mode - "use strict"; otherwise 'sloppy mode'

Linting tools - used to ensure code follows good practice; JS Lint, JS Hint, ES Lint

Browser Debugging - alert(), console.log, debugging tools

Exception Handling - Try, catch, finally

Test-driven Development (TDD) - writing tests that ail, then code that passes the test, then refactoring

Jest framework - can be used to test code
