Chapter 12 - Object Oriented Programming

Object-oriented programming (OOP)

Main Concepts:
Encapsulation - inner workings kept hidden; only essential exposed to end user
Polymorphism - the same process can be used for different objects;  allows objects to override shared methods with a more specific implementation
Inheritance - taking features of one object and then adding new features

Constructor function - create object instances (new); parenthesis not required if without default arguments
Class declaration - 'class' keyword - can be used in place of constructor functions
this - inside constructor/class refers to the object returned by the function; value not retained in nested functions
prototype - where all instances of a class/constructor function inherits properties and methods
getter/setter function - make properties/methods public
Monkey-patching - adding methods to built-in objects' prototypes
Mixin method - used in adding properties/methods from other objects w/o creating an inheritance chain
Composition over inheritance - building block objects than inheriting all properties and methods of the parent class
